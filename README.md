# Tracker-habits — трекер привычек

Кроссплатформенное настольное приложение для ежедневного учёта привычек с графическим интерфейсом. Позволяет добавлять привычки, отмечать их выполнение за любой день, отслеживать серии выполнения (streak), оценку вероятности срыва и тепловую карту активности в двух форматах. Данные сохраняются между запусками.

Проект выполнен в рамках учебной ознакомительной практики.

## Возможности

- Добавление и удаление привычек прямо в окне.
- Отметка выполнения за любой день: переключение даты кнопками ◀ ▶ и кнопкой «Сегодня».
- Автосохранение данных между запусками (`~/.habit_tracker/data.json`), устойчивость к повреждённому файлу данных.
- Алгоритмическое ядро: текущая и максимальная серия выполнения, скользящее среднее, оценка вероятности срыва по взвешенному скользящему среднему.
- Тепловая карта активности в двух форматах — линейном (30 дней) и недельном (календарь) — с числами дней и подписями месяцев.
- Таблица недельной и месячной статистики с подсветкой привычек в зоне риска.
- Генерация демонстрационных (синтетических) данных за неделю или месяц.
- Экспорт отчёта в CSV, корректно открывающийся в Excel.

## Запуск из исходного кода

### Требования

- **Python 3.8 или новее.**
- Модуль **tkinter** (графический интерфейс):
  - Windows и macOS — входит в стандартную установку Python, ставить ничего не нужно;
  - Linux — на части дистрибутивов ставится отдельным пакетом:

    ```bash
    sudo apt install python3-tk        # Debian / Ubuntu
    sudo dnf install python3-tkinter   # Fedora
    sudo pacman -S tk                  # Arch
    ```

Сторонние библиотеки не требуются — используется только стандартная библиотека Python.

Проверить доступность tkinter:

```bash
python -c "import tkinter; print('tkinter OK')"
```

### Запуск

```bash
python tracker-habits-final.py
```

На некоторых системах команда называется `python3`:

```bash
python3 tracker-habits-final.py
```

### Сборка исполняемого файла вручную

На любой из трёх ОС результат для этой ОС собирается одной командой:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name tracker-habits tracker-habits-final.py
```

Готовый файл появится в папке `dist`. PyInstaller не кросс-компилирует: сборка под Windows выполняется на Windows, под macOS — на macOS (либо используйте GitHub Actions, см. выше).

## Кроссплатформенность

Приложение работает на Windows, macOS и Linux без изменений кода:

- интерфейс построен на tkinter и одинаков на всех ОС;
- пути к файлам формируются через `pathlib` и `Path.home()`, поэтому файл данных корректно располагается в домашней папке на любой ОС;
- нет сторонних зависимостей, требующих компиляции.

## Как пользоваться

### Отметка привычек за день

Слева в блоке «Отметки за день» перечислены привычки с галочками. Галочка ставит или снимает отметку выполнения за выбранный день. Рядом с каждой привычкой показан прогноз `P=…%` — вероятность срыва.

### Навигация по дням

- **◀ / ▶** — предыдущий или следующий день (в пределах последних 30 дней);
- **Сегодня** — вернуться к текущей дате;
- выбранный день подсвечивается оранжевой рамкой на тепловой карте.

### Добавление и удаление привычек

- Введите название в поле внизу левого блока и нажмите **«+ Добавить»** (или Enter). Пустые и повторяющиеся названия не принимаются.
- Кнопка **✕** справа от привычки удаляет её вместе со всей историей (с подтверждением).

### Тепловая карта активности

Переключатель над картой выбирает формат:

- **Линейная (30 дней)** — строки соответствуют привычкам, столбцы — дням; сверху подписи месяцев и числа дней, нижняя строка «ИТОГО/день» показывает агрегированную активность за день;
- **Недельная (календарь)** — строки соответствуют дням недели (Пн…Вс), столбцы — неделям; число дня выводится внутри клетки, а её цвет (оттенки зелёного) отражает долю выполненных привычек за этот день.

### Демо-данные (синтетика)

Кнопки в верхней панели:

- **Синтетика: неделя** — добавляет демонстрационные данные за 7 дней;
- **Синтетика: месяц** — за 30 дней.

Данные дозаполняются к текущим привычкам (с подтверждением). Для реальной работы синтетика не нужна — достаточно добавить свои привычки и отмечать их каждый день.

### Экспорт отчёта

Кнопка **«Экспорт (CSV)»** сохраняет журнал выполнения за 30 дней в CSV-файл: строки — даты, столбцы — привычки, значения 0/1.

Файл записывается в кодировке **UTF-8 с BOM** и с разделителем **«;»**, поэтому открывается в Excel (в том числе с русскими региональными настройками) сразу с корректной кириллицей и разбивкой по столбцам — без мастера импорта.

## Где хранятся данные

Все привычки и отметки автоматически сохраняются в файл:

```text
~/.habit_tracker/data.json
```

где `~` — домашняя папка пользователя (`C:\Users\Имя` в Windows, `/home/имя` в Linux, `/Users/имя` в macOS).

Сохранение происходит после каждого изменения и при закрытии окна, загрузка — при следующем запуске. Если файл повреждён или имеет неверную структуру, приложение не падает, а запускается с пустым журналом. Чтобы начать с чистого листа, просто удалите этот файл.

Формат файла — JSON; в журнал записываются только выполненные дни:

```json
{
  "habits": ["Зарядка", "Чтение"],
  "log": {
    "Зарядка": { "2026-06-16": true }
  }
}
```

## Алгоритмическое ядро

- **Текущая серия (streak)** — число подряд идущих выполненных дней, считая с последнего дня периода; прерывается на первом невыполненном дне.
- **Максимальная серия** — самая длинная непрерывная серия выполнения за весь период.
- **Скользящее среднее** — средняя доля выполнения за последние 7 дней (используется в колонке «Неделя %»).
- **Вероятность срыва:**

  ```text
  P(срыв) = 1 − взвешенное скользящее среднее за 7 дней
  ```

  Более свежие дни имеют больший вес (линейные веса): чем чаще пропуски в последние дни, тем выше вероятность срыва. Значения `P ≥ 0.5` помечаются как риск — красным цветом в таблице статистики и рядом с привычкой.

Ядро отделено от интерфейса: расчётные функции не зависят от tkinter и могут тестироваться автономно.
